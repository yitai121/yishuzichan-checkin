## Task 3 【P0】大屏页 `/screen` 路由与 `/api/screen` 数据接口

### 设计稿（不要改）
- 文件路径（设计稿源文件）：`screen.html`（已交付，700 行，1920×1080 横屏）
- 设计稿在线预览：`https://www.coze.cn/s/3ENOxJ3Dwuc/`
- **直接把 screen.html 复制到项目 public 或 static 目录**，挂在 `/screen` 路由下，不要动里面的 HTML/CSS/JS

### 后端要做的接口

**GET `/api/screen`**
- 无需鉴权
- 200ms 内响应
- 可以加 1 秒缓存

返回 JSON 结构（严格遵循）：
```json
{
  "count": 87,
  "total": 300,
  "rate": 29.0,
  "ratePerMin": 4.5,
  "etaTime": "13:58",
  "recent": [
    {"name": "陈致远", "org": "亿数生态", "checkedAt": "13:42:18"},
    {"name": "林雨晴", "org": "国文汇通", "checkedAt": "13:41:55"}
  ]
}
```

字段说明：
- `count` 已签到人数
- `total` 应到总人数（来自参会名单）
- `rate` 百分比（count / total * 100，保留 1 位小数）
- `ratePerMin` 最近 5 分钟内每分钟签到速率
- `etaTime` 按当前速率推算全员到齐的时刻，`HH:MM` 字符串；速率为 0 时返回 `"—"`
- `recent` 最近 8 条签到记录，按时间倒序

### 前端要做的事
1. 把 `screen.html` 的 `fetch('/api/screen')` 轮询代码确认存活（设计稿已包含，不要重写）
2. 倒计时基准时间硬编码为：`new Date('2026-06-27T14:00:00+08:00')`
3. 真实签到一次后 3 秒内大屏：数字跳 +1 + 顶部金线扫描 + 撒花 + 欢迎语条出现

### 验收
1. 浏览器全屏（F11）打开 `https://f59962zb6q.coze.site/screen`，分辨率 1920×1080
2. 看到衬线大数字 + 米白底 + 蓝紫光晕 + 倒计时 6/27 14:00（**这是字体核心检查点，必须是 Fraunces 衬线，不是普通黑体**）
